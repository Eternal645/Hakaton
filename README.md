# Hakaton / T2.beta — планирование графиков и смен (demo)

**T2.beta** — демо веб‑приложение для планирования смен: сотрудники заполняют график на 2 недели вперёд, руководители видят нагрузку/отклонения и подтверждают план/факт, есть отчёты и экспорт данных.  
Проект сделан в стиле “всё в одном”: Node.js сервер (Express) + статический фронтенд (HTML/CSS/JS) + хранение данных в `db.json`.

---

## Возможности (что умеет приложение)

- **Планирование на 2 недели вперёд**: сотрудник заполняет смены заранее.  
- **Руководитель**: контроль нагрузки, просмотр отклонений и подтверждение “план/факт”.  
- **Аналитика**: тепловая карта нагрузки команды.  
- **Отчёты**: экспорт (CSV/JSON), выгрузка по периодам.

---

## Стек и архитектура

- **Backend:** Node.js + Express, включён CORS, сервер раздаёт статику из `public/`.  
- **Frontend:** чистый HTML/CSS/JS (лежит в `public/`), обращается к backend по REST (`/api/*`).  
- **Хранилище:** файл `db.json` в корне проекта (JSON‑“база”).

Архитектурно это один процесс: `server.js` и фронтенд раздаются одним сервером.

---

## Структура репозитория

- `server.js` — Express‑сервер и API.
- `public/`
  - `index.html` — основной UI (SPA-подобный, с переключением страниц внутри).
  - `api.js` — клиент для вызова API.
  - `style.css` — стили.
- `db.json` — данные (пользователи/организации/смены/уведомления).
- `package.json` — зависимости и npm‑скрипты.
